# 实例配置可视化保存保留额外字段说明

## 变更背景

实例管理中的配置文件可视化编辑以前会按照配置模板重新生成配置内容。
当实际配置文件里存在模板中未声明的额外键值时，这些键值会在保存后被覆盖或丢失。

## 当前行为

现在保存实例配置时会先读取现有配置文件，再只把模板中对应的字段写回去：

- 模板内字段会按当前表单值更新
- 模板外的额外键值会原样保留
- 模板外的额外 section 或根级字段会继续保留
- 嵌套字段中未在模板声明的额外键值也会尽量保留

## 影响范围

本次调整覆盖 `GameConfigManager` 当前支持的以下格式：

- `properties`
- `configobj`
- `yaml`
- `ruamel.yaml`
- `json`
- `toml`

## 说明

本次调整的目标是避免可视化编辑误伤未建模配置项。
可视化界面仍然只展示模板中定义的字段，但保存时不会再主动清理模板外内容。
