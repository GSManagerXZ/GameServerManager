# 文件编辑器关闭缓存修复说明

## 变更概述

文件管理页中的文本编辑器在本次调整后，关闭弹窗时会同步清理当前已打开文件的内存状态，不再保留上一次编辑内容的缓存。

## 调整后的行为

- 点击编辑器弹窗右上角关闭按钮时，会清空当前已打开文件的缓存
- 点击编辑器底部“关闭”按钮时，会清空当前已打开文件的缓存
- 点击标签页关闭按钮时，仍然会只关闭当前标签对应文件
- 当最后一个标签页被关闭时，编辑器弹窗会自动关闭，且不会保留文件缓存
- 关闭弹窗后重新打开同一路径文件时，会重新从磁盘读取最新内容

## 影响说明

- 关闭编辑器弹窗后，未保存的临时修改不会再被保留
- 文件监视状态会在关闭时一并释放，避免残留监视状态影响后续重新打开

## 适用场景

- 用户关闭编辑器后重新打开同一文件，希望看到磁盘中的最新内容
- 避免因为前端内存状态残留，导致误以为文件内容没有更新
